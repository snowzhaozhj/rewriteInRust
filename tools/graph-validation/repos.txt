# 源码图差分校验 —— 钉死的真实 TS 仓库清单
#
# 格式（每行一个仓库，# 开头为注释，空行忽略）：
#   <name> <git_url> <commit_sha> <src_root>
#
# - name      ：本地工作目录名 + 报告文件名
# - git_url   ：clone 地址
# - commit_sha：精确提交 SHA（来自 git ls-remote 对应 tag，保证可复现）
# - src_root  ：相对仓库根的源码根目录（建图 / oracle 扫描的起点）
#
# 选取标准：公开、纯 TS、结构干净（无 monorepo、几乎无 .d.ts 噪声）、
# 文件量级覆盖小/中（十几 ~ 两百多文件），依赖结构有代表性（含菱形、循环）。
#
# Oracle 工具版本（钉死，见 oracle/package.json）：
#   dependency-cruiser@16.10.4   （主；16.10.0 与 node v24 的 node:fs R_OK 不兼容，故用 .4）
#   dpdm@3.14.0                  （交叉验证）
#   typescript@5.6.3             （oracle 解析依赖）
